Abstract

Post-disaster microgrid formation increasingly depends on mobile energy storage, electric vehicles and UAV-assisted communication. Weak communication creates an execution gap because a feasible network action can still lack valid packets, route evidence, localization confidence or control authority. SAFE-GPINN addresses this gap by applying EWMAC admission before neural ranking. In five-seed closed-loop experiments, it restores all eight released feeder faults, reaches reward 415.23, reduces shed energy relative to RH-MPC by 103.14 kWh and improves delay by 4.45 ms under the reconstructed PTIN case.

PTIN State

The state joins PDN energization, UTN travel time, CN packet feedback, mobile-resource energy and sparse weak-communication memory.

s_t = {x_t^P, x_t^T, x_t^C, e_t^M, b_t}

EWMAC Shield

The shield rejects candidates before ranking. A high neural score cannot override failed packets, route conflict or negative control margin.

chi_t^EWMAC(a) = chi_t^P chi_t^T chi_t^C chi_t^Z chi_t^B chi_t^U

Objective

The closed-loop objective rewards restored critical load and penalizes shed energy, time, unsafe attempts, communication delay and controller unavailability.

max sum_t R_t = R_load - R_shed - R_time - R_comm - R_block

SAFE-GPINN details

Algorithm Modules

1

Generate restoration, relay, predeployment and waiting candidates from the source PTIN state.

2

Apply EWMAC hard gates and record blocked reasons.

3

Estimate physical residual margins with Graph-PINN features over admitted actions.

4

Rank admitted candidates using return score, pairwise order regularization and prefix feasibility.

5

Use controlled relay predeployment and late reward gating to improve communication and terminal efficiency.
