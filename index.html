Actionability-Shielded Fleet Dispatch for Microgrid Formation in Weak-Communication Power-Transportation-Information Networks

SAFE-GPINN studies disaster recovery in a coupled power, transportation and communication network. It converts physically feasible restoration candidates into executable actions by checking route reachability, packet evidence, controller availability and mobile-resource state before neural ranking.

Abstract

Post-disaster microgrid formation increasingly depends on mobile energy storage, electric vehicles and UAV-assisted communication. Weak communication creates an execution gap because a network-feasible action can still lack valid packets, route evidence, localization confidence or control authority. SAFE-GPINN closes this gap by applying an EWMAC actionability shield before Graph-PINN ranking and closed-loop execution. In five-seed experiments, it restores all eight released feeder faults and reaches reward 415.23 while maintaining high packet delivery under the reconstructed PTIN case.

Research object

Coupled Power, Transportation and Information Network

The object is a post-disaster PTIN restoration case. The PDN supplies loads, the UTN constrains mobile-resource travel, and the CN constrains packets, delay and controller authority. Cross-layer dependencies decide whether an apparently feasible restoration action can be executed.

PDN buses, feeders, released faults UTN junctions, roads, travel times CN nodes, links, packet feedback Fleet MESS, UGV and UAV resources

Complete modeling process

Modular Mathematical Model

Sets and Indices

  • G^P = (V^P, E^P) distribution network
  • G^T = (V^T, E^T) transportation network
  • G^C = (V^C, E^C) communication network
  • m in M mobile resource
  • k in K_t released restoration task
  • t in T closed-loop decision segment

State and Decision Variables

  • z^P_{i,t} PDN bus energization
  • x^P_{e,t} feeder switching and restoration state
  • r_{m,t} mobile-resource route
  • q_{c,t}, d_{c,t} packet delivery and delay
  • u_{m,k,t} assignment of resource m to task k
  • a_t selected restoration action

Layered State

The closed-loop state contains energized buses and feeders, travel time, mobile-resource energy, relay coverage, packet feedback, controller status and the sparse memory of failed action attempts.

s_t = {x^P_t, z^P_t, r_t, e^M_t, q_t, d_t, b_t}

Actionability Shield

A candidate action is executable only when power feasibility, route feasibility, packet evidence, controller authority, memory state and fleet conflict gates are all satisfied.

A_t^safe = {a in A_t | chi^P chi^T chi^C chi^Z chi^B chi^U = 1}

Constraint Catalogue

The table lists the constraints shown in the model so the advisor can see which physical and cyber conditions are considered.

Block Constraint Meaning
Power z^P_{i,t} in {0,1} A load bus is either energized or not energized.
Power 0 <= p^{serve}_{i,t} <= p^d_i z^P_{i,t} Served load cannot exceed demand and must be zero when the bus is not energized.
Power V_min <= V_{i,t} <= V_max Voltage must stay within the accepted operating range.
Power |S_{e,t}| <= S_bar_e x^P_{e,t} Line loading is limited by feeder capacity and switching state.
Mobile power 0 <= p^M_{m,t} <= P_bar_m y_{m,t} A mobile resource can inject power only after it reaches an energized task site.
Mobile power E_{m,t+1} = E_{m,t} - e^{travel}_{m,t} - e^{serve}_{m,t} Battery energy accounts for travel and service consumption.
Traffic tau_{m,k,t} = tau^{SUMO}_{r_{m,k},t} Travel duration is obtained from the UTN route and SUMO or SUMO-derived TraCI feedback.
Traffic r_{m,t} in R^T_{m,t} The selected route must remain reachable under road damage and congestion.
Communication q_{c,t} >= q_min The control packet delivery rate must support the selected action.
Communication d_{c,t} <= d_max The communication delay must stay below the action deadline.
Control z^{ctrl}_{i,t} = 1 The relevant controller must be powered and reachable.
Fleet sum_k u_{m,k,t} <= 1 Each mobile resource can serve at most one task in a decision segment.
Actionability chi^P chi^T chi^C chi^Z chi^B chi^U = 1 A candidate passes only when all layer gates and memory gates are satisfied.

Total Objective

The objective maximizes critical load restoration and penalizes shed energy, terminal time, unsafe attempts, communication loss and controller unavailability.

max pi sum_t [ alpha_L L^{serve}_t - alpha_E E^{shed}_t - alpha_T Delta t_t - alpha_C(1 - q_t) - alpha_D max(d_t - d_max, 0) - alpha_B B_t + alpha_F F_t ]

SAFE-GPINN details

Algorithm Modules

1

Candidate generation builds restore, relay, predeployment and waiting actions from released faults, mobile fleet locations and current PTIN state.

2

EWMAC admission tests power, traffic, communication, controller, memory and fleet gates before any neural ranking is allowed.

3

Graph-PINN residual ranking estimates physical margin, load recovery gain and cross-layer action risk over the admitted set.

4

Pairwise order regularization and prefix feasibility keep high-score actions consistent with executable restoration order.

5

Closed-loop execution updates PTIN state, records blocked reasons and triggers relay predeployment when communication is the bottleneck.

Action Selection Rule

a_t^* = argmax_{a in A_t^safe} [ S_pi(s_t, a) + lambda_R Z_phi(s_t, a) + lambda_K kappa_t(a) ]

Data and simulator design

Layered Data Support for PTIN Experiments

Reconstructed PTIN Case

PDN, UTN and CN nodes and edges are stored as auditable tables. The case also records cross-layer dependencies, failure-release steps and mobile fleet resources.

Traffic and SUMO

The UTN graph provides the route substrate. SUMO or SUMO-derived TraCI feedback supplies travel time, speed, waiting time and occupancy, which determine action duration and traffic feasibility.

Communication and ns-3

CN topology and packet replay are converted into packet delivery rate, delay and powered-controller rate. UAV relay and dual-channel actions are evaluated through these quantities.
